# Kochwelt – Project Documentation

This document describes how the team organizes the development of the Kochwelt project.

---

## Project Scope

The goal of this project is to build a simple recipe website.

The minimum version of the project should include:

- Homepage
- Recipe overview page
- Individual recipe page
- Navigation menu
- Footer
- Responsive layout

Additional improvements may be added if time allows.

---

## Team Workflow

All team members work on HTML, CSS, and JavaScript together.

The goal is that everyone gains experience with all core technologies used in the project.

Before starting larger changes, the team briefly discusses the task to make sure everyone understands the current project state.

Team members communicate regularly to share progress and coordinate upcoming tasks.

---

## Git Workflow

The project uses a simple Git workflow.

Main branch:
- `main` contains the stable version of the project.

Development:
- New features are created in feature branches.

Examples:
- `feature/navigation`
- `feature/recipe-cards`
- `feature/footer`

After completing a feature, the changes are reviewed by the team and merged into the `main` branch.

---

## Task Organization

Project tasks are managed using a shared Trello project board.

Tasks are discussed and assigned during team meetings.

---

## Communication

The team communicates regularly to coordinate development.

Important changes or structural decisions should be discussed briefly with the team before implementation.

This helps ensure that everyone understands the direction of the project.

---

## File Organization

The project uses the following folder structure:

/html - html files
/css – stylesheets  
/js – JavaScript files  
/assets – images, fonts and media  
/docs – project documentation

---

## Learning Goal

This project is primarily intended as a learning experience for working collaboratively on a web development project using Git and basic web technologies.
